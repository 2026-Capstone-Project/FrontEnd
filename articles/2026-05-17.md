### [Yeonjin Kim] 일정 기본 시간 범위 보정 및 검색 최적화

Yeonjin Kim 님은 일정 입력 폼에서 발생할 수 있는 시간 계산 오류를 방지하기 위해 보정 로직을 강화했습니다. 기존의 단순 날짜 기반 포맷팅에서 벗어나 분(Minute) 단위의 변환 로직을 도입하여 시간 범위 계산의 정확도를 높였습니다.

**src/shared/hooks/form/useScheduleFormFields.ts**
```typescript
const toMinutes = (value?: string) => {
  if (!value) return null
  const [hour, minute] = value.split(':').map((item) => Number.parseInt(item, 10))
  if (Number.isNaN(hour) || Number.isNaN(minute)) return null
  return hour * 60 + minute
}

const formatTimeFromMinutes = (value: number) => {
  // 분 단위 데이터를 다시 HH:mm 포맷으로 변환하는 로직
  const hours = Math.floor(value / 60);
  const minutes = value % 60;
  return `${pad2(hours)}:${pad2(minutes)}`;
}
```

또한, 일정 제목 입력 시 서버 부하를 줄이고 UX를 개선하기 위해 쓰로틀링(Throttle) 기법을 적용했습니다. 제목 검색 호출 간격을 300ms로 제한하여 불필요한 API 요청을 최적화했습니다.

### [Yeonjin Kim] 반복 수정 범위 상수화 및 타입 시스템 정립

반복 일정 수정 시 사용되는 범위(Scope) 문자열들이 코드 곳곳에 산재해 있어 유지보수가 어려웠던 점을 개선했습니다. `RECURRENCE_EVENT_SCOPE`와 `RECURRENCE_TODO_SCOPE`를 중앙 집중형 상수로 관리하고, 이를 기반으로 타입을 재정의하여 런타임 에러 가능성을 원천 차단했습니다.

**src/shared/constants/recurrenceScope.ts**
```typescript
export const RECURRENCE_EVENT_SCOPE = {
  THIS_EVENT: 'THIS_EVENT',
  THIS_AND_FOLLOWING_EVENTS: 'THIS_AND_FOLLOWING_EVENTS',
  THIS_AND_FOLLOWING: 'THIS_AND_FOLLOWING',
} as const

export const RECURRENCE_TODO_SCOPE = {
  THIS_TODO: 'THIS_TODO',
  THIS_AND_FOLLOWING: 'THIS_AND_FOLLOWING',
} as const

export type RecurrenceEventScope =
  (typeof RECURRENCE_EVENT_SCOPE)[keyof typeof RECURRENCE_EVENT_SCOPE]
```

이 변경사항은 `useCalendarMutation`, `useScheduleSubmitFlow`, `DeleteConfirmModal` 등 프로젝트 전반의 10여 개 파일에 적용되었으며, 이를 통해 반복 일정의 수정/삭제 정책이 일관되게 유지될 수 있는 기반을 마련했습니다. 특히 API 요청 시 파라미터로 전달되는 스코프 값의 신뢰도가 크게 향상되었습니다.
