#### 1. API 및 Query Key 체계 확장
서버로부터 타이틀 히스토리를 가져오기 위한 API 함수와 TanStack Query(React Query)에서 관리할 키들을 정의했습니다. `TitleHistoryResponseDTO`를 공통 타입으로 정의하여 일정과 할 일 양측에서 재사용 가능하도록 설계되었습니다.

```typescript
// src/shared/api/calendar/api.ts 및 todo/api.ts 변경점 일부
export const getTodoTitleHistory = async (
  keyword?: string,
): Promise<TCommonResponse<TitleHistoryResponseDTO>> => {
  const { data } = await axiosInstance.get('/todos/title-history', {
    params: { keyword }
  })
  return data
}

// src/shared/types/common/common.ts
export type TitleHistoryResponseDTO = {
  titleHistory: string[]
}
```

#### 2. 성능 최적화를 위한 `useThrottledValue` 커스텀 훅 도입
사용자가 타이핑을 할 때마다 API 요청이 발생하는 것을 방지하기 위해 쓰로틀링(Throttling) 기능을 수행하는 커스텀 훅을 추가했습니다. 이는 연속적인 이벤트 발생 시 일정 시간 간격으로 마지막 값을 업데이트하여 네트워크 비용을 절감합니다.

```typescript
// src/shared/hooks/common/useThrottledValue.ts
import { useEffect, useRef, useState } from 'react'

export const useThrottledValue = <T>(value: T, delay = 150) => {
  const [throttledValue, setThrottledValue] = useState(value)
  const lastUpdatedAtRef = useRef(0)
  const timeoutRef = useRef<number | null>(null)

  useEffect(() => {
    const now = Date.now()
    const remainingTime = delay - (now - lastUpdatedAtRef.current)

    if (remainingTime <= 0) {
      setThrottledValue(value)
      lastUpdatedAtRef.current = now
    }
    // ... 중략 (timeout 로직)
  }, [value, delay])

  return throttledValue
}
```

#### 3. UI 컴포넌트의 동적 데이터 전환
기존 `TitleSuggestionInput` 컴포넌트에 하드코딩되어 있던 `defaultSuggestions`를 제거하고, `useEventTitleHistoryQuery` 및 `useGetTodoTitleHistoryQuery`를 통해 가져온 실데이터를 바인딩했습니다.

```tsx
// src/shared/ui/common/TitleSuggestionInput/TitleSuggestionInput.tsx 변경점
- const defaultSuggestions = [
-   '동아리 스터디 참여',
-   '동아리 회의 참여',
-   ...
- ]

// src/shared/ui/modal/AddSchedule/form/AddScheduleFormFields.tsx
// Throttled value를 쿼리의 인자로 전달하여 API 호출 최적화
const throttledKeyword = useThrottledValue(currentTitle)
const { data: historyData } = useEventTitleHistoryQuery(throttledKeyword, !!throttledKeyword)
```

### 기술적 분석 요약
1.  **데이터 무결성 및 재사용성**: `TitleHistoryResponseDTO`를 공통 타입으로 분리하여 일정(Calendar)과 할 일(Todo) 도메인 간의 일관성을 유지했습니다.
2.  **UX 최적화**: `useThrottledValue`를 통해 입력창의 반응성을 유지하면서도 서버 부하를 최소화하는 전략을 취했습니다.
3.  **선언적 데이터 페칭**: `query-key-factory`를 활용해 쿼리 키를 구조적으로 관리함으로써 유지보수성을 높였습니다.
