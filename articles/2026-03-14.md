# [2026년 03월 14일] 일정 및 투두 장소 검색 기능 고도화 및 카카오맵 연동 최적화

오늘 팀 "Calio"는 일정 및 투두 관리의 핵심 기능 중 하나인 '장소 추가' 기능을 집중적으로 개선했습니다. 특히 카카오맵 SDK 연동 과정에서 발생할 수 있는 타입 안정성 문제를 해결하고, 모달의 렌더링 위치(Portal)를 컨텍스트에 따라 유연하게 대응할 수 있도록 구조를 개편했습니다. 사용자 경험(UX) 측면에서는 지도 로딩 상태 안내와 검색 UI의 시각적 완성도를 높이는 데 주력했습니다.

코드 리뷰에서 논의된 사항을 바탕으로 프로젝트 환경을 정비했습니다. `.eslintcache`가 레포지토리에 추적되지 않도록 `.gitignore`를 수정하였으며, 카카오맵 API 사용 시 타입 추론을 위해 `kakao.maps.d.ts` 의존성을 추가했습니다.

#### 2. 일정 장소 입력 모달 동작 및 포탈(Portal) 로직 개선 (Feat)
기존의 고정된 포탈 방식에서 벗어나, 모달이 호출되는 환경(모달형 vs 인라인형)에 따라 렌더링 위치를 동적으로 결정할 수 있도록 `modalWrapperElement`를 도입했습니다. 이를 통해 장소 검색 창이 부모 컨테이너 내부 혹은 뷰포트 전체 중 적절한 위치에 배치되도록 최적화했습니다.

```typescript
// src/shared/ui/modal/AddSchedule/styles/index.style.ts
// 배치 방식에 따른 스타일 분기 처리
export const SearchPlacePortal = styled.div<{ $placement?: 'container' | 'viewport' }>`
  position: ${({ $placement }) => ($placement === 'container' ? 'absolute' : 'fixed')};
  width: min(380px, 100%);
  max-height: min(620px, calc(100vh - 24px));
  overflow-y: auto;
  // ...이하 생략
`
```

또한, `useScheduleFormAnchors` 훅에서 불필요한 상태 의존성을 제거하여 리렌더링 효율을 높였으며, 카카오맵 로딩 상태에 따른 메시지 처리를 세분화하여 사용자에게 명확한 피드백을 제공합니다.

```tsx
// src/shared/ui/modal/common/SearchPlace/SearchPlace.tsx
const mapLoadingMessage = isKakaoLoading ? '지도를 불러오는 중...' : '장소 정보를 불러오는 중...'

return (
  <S.Wrapper>
    <S.InputForm role="search" aria-label="장소 검색">
      {/* 로딩 상태 및 검색 입력부 */}
    </S.InputForm>
  </S.Wrapper>
)
```

#### 3. UI/UX 스타일링 및 인터랙션 정교화 (Style)
장소 검색 입력창의 보더 색상을 조정하고, 로딩 시 시각적 효과를 위한 `spin` 애니메이션을 추가했습니다. 불필요한 확인 버튼을 제거하고 검색 인터랙션을 간소화하여 사용자가 더 빠르게 장소를 선택할 수 있도록 개선했습니다.

```typescript
// src/shared/ui/modal/common/SearchPlace/SearchPlace.style.ts
const spin = keyframes`
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
`

export const SearchInput = styled.input`
  width: 100%;
  border-radius: 12px;
  padding: 12px 84px 12px 14px;
  border: 1px solid #acacac; /* 시인성 개선 */
  background-color: ${theme.colors.white};
  color: ${theme.colors.textColor2};
`
```

### 시니어 개발자 총평
오늘 작업의 핵심은 **"유연한 UI 구조 확보"**입니다. 단순히 기능을 구현하는 것에 그치지 않고, `Portal`의 위치를 `container`와 `viewport`로 분리하여 정의함으로써 향후 다양한 레이아웃에서 '장소 검색' 컴포넌트를 재사용할 수 있는 기반을 마련했습니다. 또한 TypeScript 정의 추가를 통해 카카오맵 연동 시 발생할 수 있는 런타임 에러를 사전에 방지한 점이 고무적입니다. 내일은 검색 결과 리스트의 가독성 개선과 실제 일정 데이터 저장 로직과의 최종 연결을 마무리할 예정입니다.
