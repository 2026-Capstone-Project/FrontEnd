### [Yeonjin Kim] 일정 CRUD 서버 연동 및 대규모 리팩토링

Yeonjin Kim 개발자는 기존의 목업 데이터를 제거하고 실제 서버 API와의 연동을 완료했습니다. 특히 비대해진 폼 컴포넌트를 기능별로 분리하고, 서버의 반복 일정(Recurrence) 스펙을 프런트엔드 타입 시스템에 완벽하게 이식했습니다.

#### 1. 서버 스펙 기반의 타입 재정의 및 API 추상화
서버의 `Event` 객체와 반복 설정(`RecurrenceGroup`)을 `shared` 영역으로 이동하여 전역적인 타입 안전성을 확보했습니다. 또한 `@lukemorales/query-key-factory`를 도입하여 TanStack Query의 키 관리를 체계화했습니다.

```typescript
// src/shared/api/queryKey/queryKeys.ts
export const calendarKeys = createQueryKeys('calendar', {
  events: (startDate: string, endDate: string) => ({
    queryKey: [{ startDate, endDate }],
    queryFn: () => getEvents({ startDate, endDate }),
  }),
  detail: (eventId: number, occurrenceDate: string) => ({
    queryKey: ['event', { eventId, occurrenceDate }],
    queryFn: () => getDetailEvent(eventId, occurrenceDate),
  }),
});
```

#### 2. AddScheduleForm의 원자적 분리 (Atomic Refactoring)
수백 줄에 달하던 `AddScheduleForm`을 `AddScheduleFormFields`, `AddScheduleFormContent`, `AddScheduleFormConfirmModals` 등으로 분리했습니다. `FormProvider`를 활용해 상태를 공유하며, 각 컴포넌트가 단일 책임 원칙(SRP)을 준수하도록 개선했습니다.

#### 3. 반복 일정 선택 및 렌더링 버그 수정
반복 일정 중 하나를 선택했을 때 전체가 선택되는 문제를 해결하기 위해, ID와 시작 시간을 조합한 고유 키(`getEventOccurrenceKey`)를 도입했습니다.

```typescript
// src/features/Calendar/utils/helpers/dayViewHelpers.ts
export const getEventOccurrenceKey = (event: CalendarEvent) =>
  `${event.id}_${moment(event.start).format('YYYY-MM-DDTHH:mm')}`;
```

#### 4. CI/CD 및 개발 환경 개선
- `.github/workflows/daily_article.yml`: 개발 일지 생성 후 PR을 날리는 방식에서 직접 Push하는 방식으로 워크플로우를 간소화했습니다.
- `pull_request_template.md`: Copilot을 활용한 코드 리뷰 지침을 추가하여 리뷰 퀄리티를 표준화했습니다.

## 🌿 Branch: feature/#28

### [yujin5959] 설정 페이지 API 연동 및 비즈니스 로직 분리

yujin5959 개발자는 사용자 설정(알림 타이밍, 브리핑 시간, 기본 뷰 등)을 서버와 주고받는 기능을 구현하고, UI 컴포넌트에서 API 호출 로직을 분리하는 리팩토링을 진행했습니다.

#### 1. 설정 관련 API 및 타입 정의
사용자의 서비스 이용 경험을 결정짓는 설정 데이터 타입을 정의하고 API 객체를 구성했습니다.

```typescript
// src/shared/types/settings/settings.ts
export type ReminderTiming =
  | 'FIVE_MINUTES' | 'FIFTEEN_MINUTES' | 'THIRTY_MINUTES'
  | 'ONE_HOUR' | 'TWO_HOURS' | 'ONE_DAY';
export type CalendarView = 'MONTH' | 'WEEK' | 'DAY';

export interface UserSettings {
  dailyBriefingEnabled: boolean;
  dailyBriefingTime: string;
  reminderTiming: ReminderTiming;
  defaultView: CalendarView;
}
```

#### 2. useSettingsMutation 커스텀 훅 도입
설정 변경 로직을 `useSettingsMutation`으로 추상화하여 `SettingsPage` 컴포넌트의 복잡도를 낮추고, 변경 성공 시 `queryClient.invalidateQueries`를 통해 최신 데이터를 유지하도록 처리했습니다.

```typescript
// src/shared/hooks/useSettingsMutation.ts
export const useSettingsMutation = () => {
  const queryClient = useQueryClient();
  const navigate = useNavigate();
  
  return useMutation({
    mutationFn: (settings: Partial<UserSettings>) => SettingsAPI.updateSettings(settings),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['settings'] });
    },
  });
};
```
