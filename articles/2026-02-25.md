#### 1. 할 일 중요도(Priority) 시스템 도입 및 API 연동
사용자가 할 일의 우선순위를 명확히 관리할 수 있도록 중요도 선택 기능을 추가했습니다. `PriorityType`을 정의하고, 이를 폼 스키마 및 UI에 녹여내어 데이터의 일관성을 확보했습니다.

*   **기술적 분석:** `yup` 스키마를 통해 `HIGH`, `MEDIUM`, `LOW` 값을 검증하며, `useAddTodoForm` 및 `useTodoFormFields` 훅을 확장하여 상태 관리를 통합했습니다.

```typescript
// src/shared/schemas/todo.ts 변경 사항
const todoPriority = yup
  .mixed<PriorityType>()
  .oneOf(['HIGH', 'MEDIUM', 'LOW'], '유효하지 않은 우선순위입니다.')
  .required('우선순위를 선택하세요.')
```

#### 3. 반복 일정 수정 및 제출 로직의 안정화
반복되는 할 일을 수정할 때 서버 데이터와 클라이언트 상태 간의 동기화 문제를 해결했습니다. 특히 '낙관적 업데이트(Optimistic Update)' 이후 서버 응답에 따라 상태를 초기화하는 로직을 강화했습니다.

*   **기술적 분석:** `useTodoCardActions`에서 `onSuccess` 콜백을 통해 서버 데이터를 '단일 진실 원천(Single Source of Truth)'으로 확정 짓도록 처리하여 데이터 무결성을 높였습니다.

```typescript
// src/features/Todo/hooks/useTodoCardActions.ts
onSuccess: () => {
  // 성공 이후에는 서버 데이터(isCompleted)를 단일 진실 원천으로 사용합니다.
  setOptimisticSelected(null)
```
