## 🌿 Branch: develop

### [Yujin Ji] UI/UX 개선 및 브랜드 에셋 반영 (PR #62)

**Yujin Ji**님은 이번 작업에서 서비스의 시각적 정체성을 확립하고, 기존 UI에서 발견된 미세한 스타일 오류들을 수정했습니다. 주요 변경 사항에 대한 기술적 분석은 다음과 같습니다.

#### 1. 브랜드 아이덴티티 반영 (Favicon 교체)
기본 설정이었던 Vite 로고를 삭제하고, Calio만의 고유한 브랜드 이미지가 담긴 `favicon.svg`를 적용했습니다. 이는 서비스의 전문성을 높이는 필수적인 작업입니다.

```html
<!-- index.html -->
<head>
  <meta charset="UTF-8" />
  <meta name="Calio" content="말하는 대로 움직이는 일정" />
- <link rel="icon" type="image/svg+xml" href="/vite.svg" />
+ <link rel="icon" type="image/svg+xml" href="/favicon.svg" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>말하는 대로 움직이는 일정, Calio</title>
</head>
```

#### 2. 인증 컴포넌트 스타일 정교화
`LoginCard` 내의 소셜 로그인 버튼 텍스트가 상속된 컬러나 방문 후 상태(`:visited`)에 따라 변하지 않도록 `black`으로 명시하여 시인성을 확보했습니다.

```typescript
// src/features/Auth/components/LoginCard/LoginCard.style.ts
export const ButtonText = styled.span`
  flex: 1;
  text-align: center;
- color: inherit;
+ color: black;
  text-decoration: none;
  &:visited,
  &:active,
  &:link {
-   color: inherit;
+   color: black;
    text-decoration: none;
  }
`
```

#### 4. 헤더 컴포넌트 기능 고도화 및 고정 레이아웃 적용
사용자가 페이지를 스크롤하더라도 내비게이션이 가능하도록 `sticky` 포지셔닝을 적용했습니다. 또한, 모바일 환경을 고려한 메뉴 토글 상태 관리 로직을 추가하고, 로고 클릭 시 메뉴가 자동으로 닫히도록 UX를 개선했습니다.

오늘의 작업으로 Calio의 전반적인 UI 안정성이 크게 향상되었습니다. 특히 헤더의 `sticky` 적용과 `z-index` 설정은 향후 추가될 복잡한 콘텐츠 레이어와의 간섭을 방지하는 중요한 기반이 될 것입니다. 시니어 개발자로서 이러한 세밀한 UI/UX 개선이 서비스의 전체적인 퀄리티를 결정짓는 핵심 요소임을 다시 한번 강조하며 오늘 일지를 마칩니다.
