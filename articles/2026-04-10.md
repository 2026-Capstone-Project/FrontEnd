### [Yujin Ji] 모바일 사이드바 z-index 레이어링 수정
유진 님은 모바일 뷰에서 사이드바(Nav)를 열었을 때 로고나 메뉴 버튼이 부적절하게 가려지거나 위에 올라오는 문제를 해결했습니다. `z-index` 계층을 재설정하여 UI 요소 간의 시각적 우선순위를 명확히 했습니다.

```typescript
+ z-index: 1001;
```

### [Yeonjin Kim] 반복 일정 인스턴스 관리 및 모바일 UX 강화
연진 님은 반복 일정 중 특정 날짜의 이벤트만을 식별하기 위해 `occurrenceDate` 필드를 시스템 전반에 도입했습니다. 이를 통해 `updateEventRange`와 같은 유틸리티 함수가 반복 일정의 특정 인스턴스를 정확히 찾아 수정할 수 있게 되었습니다.

```typescript
// src/features/Calendar/utils/helpers/calendarPageHelpers.ts
export const updateEventRange = (
  events: CalendarEvent[],
  eventId: CalendarEvent['id'],
  start: Date,
  end: Date,
  type?: CalendarEvent['type'],
+ occurrenceDate?: CalendarEvent['occurrenceDate'],
) =>
  events.map((item) =>
-   item.id === eventId && (type ? item.type === type : true)
+   item.id === eventId &&
+   (type ? item.type === type : true) &&
+   (occurrenceDate ? item.occurrenceDate === occurrenceDate : true)
      ? { ...item, start: moment(start).format('YYYY-MM-DDTHH:mm'), ... }
```

또한, 모바일 브라우저의 툴바 유무에 따라 높이가 가변적인 문제를 해결하기 위해 `vh` 단위 대신 `dvh`(Dynamic Viewport Height) 단위를 적용하여 레이아웃 안정성을 확보했습니다.

```typescript
// src/shared/layout/MainLayout.tsx
const Wrapper = styled.div`
- min-height: 100vh;
- height: 100vh;
- max-height: 100vh;
+ min-height: 100dvh;
+ height: 100dvh;
+ max-height: 100dvh;
```

그 외에도 모바일 전용 액션 버튼(`CustomCalendarMobileActions`)과 드래프트 상태 관리를 위한 `useCalendarDraftEvent` 훅을 추가하여, 모바일 환경에서도 데스크톱과 동일한 수준의 편집 경험을 제공할 수 있게 되었습니다.
