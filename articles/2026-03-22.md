## [2026년 03월 22일] CI/CD 파이프라인 구축 및 S3 자동 배포 환경 최적화

오늘 팀 Calio는 서비스의 안정적인 배포와 운영 효율성을 높이기 위해 GitHub Actions를 활용한 CI/CD 파이프라인 구축 작업을 진행했습니다. 기존의 수동 배포 방식에서 벗어나, `main` 브랜치로의 코드 병합 시 자동으로 빌드 및 AWS S3 배포가 이루어지는 자동화 프로세스를 마련했습니다. 특히 보안이 중요한 환경 변수들을 GitHub Secrets로 관리하도록 설정하여 보안성까지 확보한 것이 오늘 작업의 핵심입니다.

## 🌿 Branch: develop

### 1. CI/CD 자동 배포 워크플로우 초기 설정 (작성자: Yujin Ji)
**Yujin Ji** 님은 프로젝트의 자동 배포를 위한 기초 뼈대인 `.github/workflows/deploy.yml` 파일을 생성했습니다. `main` 브랜치에 푸시가 발생할 때마다 `ubuntu-latest` 환경에서 Node.js 18 버전을 기반으로 의존성 설치 및 빌드가 수행되도록 정의했습니다. 이는 배포 자동화의 첫 단추로, 일관된 빌드 환경을 보장하는 중요한 작업입니다.

```yaml
name: Deploy to S3

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - name: 코드 가져오기
        uses: actions/checkout@v3

      - name: Node 설치
        uses: actions/setup-node@v3
        with:
          node-version: 18

      - name: 의존성 설치
        run: npm install

      - name: 빌드
        run: npm run build
```

### 2. 환경 변수 보안 관리 및 AWS 인증 추가 (작성자: yujin5959)
**yujin5959** 님은 빌드 과정에서 필요한 민감한 정보들을 처리하기 위해 GitHub Secrets를 연동했습니다. Vite 환경에서 필요한 서버 URL과 카카오 API 키를 `env` 섹션에 추가하여 빌드 시점에 주입되도록 구성했습니다. 또한, AWS S3에 접근하기 위한 `configure-aws-credentials` 액션을 추가하여 보안 인증 절차를 강화했습니다.

```yaml
      - name: 빌드
        run: npm run build
        env:
          VITE_SERVER_URL: ${{ secrets.VITE_SERVER_URL }}
          VITE_KAKAO_API: ${{ secrets.VITE_KAKAO_API }}

      - name: AWS 로그인
        uses: aws-actions/configure-aws-credentials@v2
        with:
          # AWS 인증 정보 설정 (Access Key, Secret Key 등)
```
