# [2026년 03월 13일] 배포 안정성 강화 및 캘린더 UX/UI 고도화

오늘 팀 Calio는 프론트엔드 배포 과정에서 발견된 잔여 버그를 신속히 해결하고, 서비스의 핵심인 캘린더 기능의 사용자 경험(UX)과 접근성을 개선하는 데 집중했습니다. 보안 취약점 해결을 위한 의존성 관리부터 UI 디테일 수정까지, 프로덕션 수준의 완성도를 높이는 작업이 전방위적으로 이루어졌습니다.

## 🌿 Branch: develop

`develop` 브랜치에서는 실 배포 환경에서 발생한 이슈 대응과 보안 패치가 주를 이뤘습니다.

### 1. 보안 취약점 대응 (작성자: Yeonjin Kim)
보안 경고를 해결하기 위해 패키지 락파일(lockfile)을 갱신하였습니다. 이는 의존성 라이브러리들의 잠재적 보안 허점을 차단하고 빌드 환경의 안정성을 보장하기 위한 필수적인 조치입니다.

### 2. 프론트 배포 버그 수정 (작성자: Yujin Ji)
배포 후 UI 깨짐 현상과 API 통신 오류를 해결했습니다.

*   **로그인 카드 UI 개선**: 소셜 로그인 버튼의 레이아웃이 깨지는 문제를 해결하기 위해 `gap`과 `padding`을 조정하고, 버튼 내 텍스트의 스타일 상속 및 링크 상태(`visited`, `active`)에 따른 색상 변화를 방지했습니다.
*   **로고 렌더링 최적화**: 로고 이미지가 일부 환경에서 흐릿하게 보이는 현상을 방지하기 위해 `image-rendering` 속성을 추가하고, GPU 가속(`transform: translateZ(0)`)을 통해 선명도를 높였습니다.
*   **API 환경 변수 정정**: Axios 인터셉터에서 토큰 재발급 시 사용되던 환경 변수명을 `VITE_API_SERVER_URL`에서 실제 정의된 `VITE_SERVER_URL`로 수정하여 인증 로직의 결함을 해결했습니다.

```typescript
// src/shared/api/axios.ts 변경 사항
- `${import.meta.env.VITE_API_SERVER_URL}/security/reissue-cookie`,
+ `${import.meta.env.VITE_SERVER_URL}/security/reissue-cookie`,
```

## 🌿 Branch: refactor/#53

캘린더의 리팩토링과 기능 고도화가 진행되었습니다. **Yeonjin Kim** 시니어 개발자가 주도하여 코드의 정합성과 사용성을 대폭 개선했습니다.

### 1. 웹 접근성(A11y) 및 UX 강화
시각 장애인 및 키보드 사용자를 위해 툴바 버튼에 `aria-label`을 추가했습니다. 또한, 캘린더 상단의 날짜 레이블을 버튼화하여 클릭 시 즉시 '오늘' 날짜로 이동할 수 있도록 UX를 개선했습니다.

```tsx
// src/features/Calendar/components/CalendarToolbar/CalendarToolbar.tsx
<button
  type="button"
  className="date-label"
  onClick={() => onNavigate('TODAY')}
  title="오늘로 이동"
  aria-label="오늘로 이동"
>
  {formattedLabel}
</button>
```

### 2. UI 디테일 및 반응형 로직 정리
*   **모달 스타일**: 삭제 확인 및 변경 사항 저장 확인 모달의 그림자(`box-shadow`)를 조정하여 시각적 깊이감을 더했습니다.
*   **포털(Portal) 로직 최적화**: 불필요하게 복잡했던 `CalendarModals`의 포털 렌더링 구조를 정리하여 컴포넌트 계층을 단순화하고 유지보수성을 높였습니다.

### 3. 캘린더 뷰 로직 수정
일간 뷰(Day View)에서 드래그 시 시간 표시가 실제 슬롯 높이와 일치하지 않던 정합성 문제를 해결했습니다. `TIMED_SLOT_CONFIG`를 통해 상수화된 값을 참조하도록 수정하여 시간 계산의 정확도를 확보했습니다.
