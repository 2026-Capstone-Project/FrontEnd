#### 1. CI/CD 및 워크플로우 개선
PR 생성 시 자동으로 프로젝트 보드에 추가하고 스프린트를 지정하는 GitHub Action을 도입하여 관리 오버헤드를 줄였습니다.

```yaml
# .github/workflows/pr-to-project.yml
name: PR을 프로젝트에 추가하고 스프린트 지정
on:
  pull_request:
    types: [opened, reopened, ready_for_review]
# ... (중략)
      - name: PR을 프로젝트에 추가하고 스프린트 설정
        uses: actions/github-script@v7
        with:
          github-token: ${{ env.PROJECT_PAT }}
```

#### 2. 전역 토스트(Toast) 시스템 도입
사용자 피드백을 일관되게 제공하기 위해 `Zustand` 기반의 `useToastStore`와 전역 `ToastViewport`를 구현했습니다. 성공, 에러, 경고 등 다양한 상태에 대응하는 UI를 갖추었습니다.

```typescript
// src/store/useToastStore.ts
export const useToastStore = create<ToastState>((set) => ({
  toast: null,
  showToast: (payload) => {
    const id = ++toastId;
    set({ toast: { ...payload, id } });
  },
  hideToast: () => set({ toast: null }),
}));
```
