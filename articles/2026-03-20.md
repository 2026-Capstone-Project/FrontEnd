# [2026년 03월 20일] 카카오맵 연동을 통한 일정 장소 관리 기능 고도화

오늘 팀 "Calio"는 서비스의 핵심 기능 중 하나인 일정 관리 경험을 한 단계 끌어올리는 작업을 진행했습니다. 사용자가 일정을 등록할 때 단순히 텍스트로 장소를 입력하는 것을 넘어, 카카오맵 API를 통해 정확한 위치를 검색하고 상세 주소를 함께 저장할 수 있는 기능을 구현했습니다. 이를 위해 프론트엔드 전반의 스키마 수정과 외부 SDK 통합, 그리고 복잡해진 모달 내 포털(Portal) 처리를 최적화하는 데 집중했습니다.

### [Yeonjin Kim] 일정 내 장소 검색 및 카카오맵 SDK 통합 분석

#### 1. 의존성 및 환경 설정 변경
카카오맵 활용을 위한 SDK와 타입 정의를 추가하고, 개발 편의를 위해 `.eslintcache`를 무시 목록에 추가했습니다.

```json
// package.json
{
  "dependencies": {
    "react-kakao-maps-sdk": "^1.2.1"
  },
  "devDependencies": {
    "kakao.maps.d.ts": "^0.1.40"
  }
}
```

#### 2. 데이터 스키마 및 API 인터페이스 확장
기존에는 장소명(`location`)만 관리했으나, 지도 API 도입에 따라 상세 주소(`address`) 필드를 새롭게 정의했습니다. `yup` 스키마를 통해 주소 데이터의 유효성 검사 로직도 강화되었습니다.

```typescript
// src/shared/schemas/schedule.ts
export const addScheduleSchema = yup.object().shape({
  eventTitle: scheduleTitle,
  eventDescription: description,
  location: yup.string().trim().max(100, '장소는 최대 100자까지 입력 가능합니다.').default(''),
  address: yup
    .string()
    .trim()
    .nullable()
    .max(255, '주소는 최대 255자까지 입력 가능합니다.')
    .default(null),
  // ...기타 필드
});
```

#### 3. 지도 검색 및 UI 로직 구현
가장 핵심적인 변경 사항은 `SearchPlace` 컴포넌트와 이를 제어하는 `useSearchPlace` 훅의 구현입니다. 카카오 로더를 통해 맵을 불러오고, 검색 결과(마커)를 관리하며, 최근 검색어 기능을 로컬 스토리지와 연동하여 사용자 편의성을 높였습니다.

*   **포털 위치 제어 최적화**: 모달 내에서 지도가 표시될 때, 뷰포트 기준(`fixed`)과 컨테이너 기준(`absolute`) 배치를 유연하게 선택할 수 있도록 스타일 구조를 개선했습니다.

```typescript
// src/shared/ui/modal/AddSchedule/styles/index.style.ts
export const SearchPlacePortal = styled.div<{ $placement?: 'container' | 'viewport' }>`
  position: ${({ $placement }) => ($placement === 'container' ? 'absolute' : 'fixed')};
  width: min(380px, 100%);
  max-height: min(620px, calc(100vh - 24px));
  overflow-y: auto;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  // ...생략
`;
```

#### 4. 폼 컨트롤러 및 훅 업데이트
`useSchedulePatch` 및 `useSchedulePatchController`를 수정하여, 일정 수정 시 기존에 저장된 장소와 주소 데이터를 폼에 초기화하고 서버에 전송할 수 있도록 인터페이스를 동기화했습니다.

```typescript
// src/shared/hooks/addSchedule/useSchedulePatchController.ts
return postEventMutation({
  title: values.eventTitle?.trim() || '새 일정',
  content: values.eventDescription ?? '',
  location: values.location?.trim() || undefined,
  address: values.address?.trim() || null,
  startTime: formatDateTime(start),
  endTime: formatDateTime(end),
  isAllDay: values.isAllday,
  // ...생략
});
```
