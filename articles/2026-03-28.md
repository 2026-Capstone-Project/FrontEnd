## 🌿 Branch: develop

`develop` 브랜치에서는 인프라 설정의 미세 조정과 신규 기능 개발, 그리고 코드 일관성을 위한 리팩토링이 주를 이루었습니다.

### 1. CloudFront 배포 스크립트 최적화 및 구문 수정 (Yujin Ji)
CloudFront 캐시 무효화(Invalidation) 명령어를 간소화하고, Distribution ID 참조 시 발생할 수 있는 구문 오류를 방지하기 위해 따옴표 처리를 추가했습니다.

```diff
-        run: aws cloudfront create-invalidation \
-          --distribution-id "${{ secrets.CLOUDFRONT_DISTRIBUTION_ID }}" \
-          --paths "/*"
+        run: aws cloudfront create-invalidation --distribution-id "${{ secrets.CLOUDFRONT_DISTRIBUTION_ID }}" --paths "/*"
```

### 2. 선제적 제안(Suggestion) API 연동 및 타입 정의 (Yujin Ji)
사용자에게 일정을 제안하는 핵심 기능의 API를 연결했습니다. `suggestionApi`를 추가하고, 관련 훅(`useSuggestions`)과 데이터 타입(`SuggestionListResponse`)을 정의하여 프론트엔드 기반을 다졌습니다.

```typescript
// src/shared/api/home/home.ts 변경 사항 일부
export const suggestionApi = {
  getSuggestions: async (): Promise<SuggestionListResponse> => {
    const res = await axiosInstance.get<SuggestionListResponse>('/suggestions')
    return res.data
  }
}
```

### 3. Import 경로 및 파일명 대소문자 수정 (Yujin Ji, yujin5959)
`suggestionkey`로 명명되었던 파일과 경로를 `suggestionKeys`로 수정하여 프로젝트 내 명명 규칙 일관성을 확보하고 대소문자 구분으로 인한 빌드 오류를 방지했습니다.

### 4. 로그인 카드 디자인 및 Query Key 리팩토링 (yujin5959)
로그인 버튼의 텍스트 색상을 명시적으로 지정하여 UI 가독성을 높였으며, React Query의 `mutation`에서 사용되는 키를 분리하여 관리 효율성을 증대시켰습니다.

### 2. 생성 모달 날짜 이동 및 데이터 동기화 로직 최적화 (김연진)
캘린더에서 이벤트를 클릭하거나 슬롯을 선택하여 모달을 열 때, 기존에 열려있던 모달의 상태를 정리하고 새로운 이벤트 데이터를 주입하는 흐름을 정리했습니다. 특히 `initialEvent`를 통해 편집 모드와 생성 모드 간의 데이터 전달을 명확히 했습니다.

### 3. 폼 필드 초기화 로직 보완 (김연진)
`useScheduleFormFields` 훅 내에서 `useEffect`를 통해 생성 모드일 때 날짜 및 종일 여부 초기값이 유연하게 적용되도록 수정하여 사용자가 날짜를 이동하며 일정을 생성할 때 발생하던 데이터 불일치 문제를 해결했습니다.
