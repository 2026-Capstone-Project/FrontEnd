### 1. 공유 일정 캘린더 아이콘 표시 (작성자: Yeonjin Kim)
공유된 일정을 사용자가 쉽게 식별할 수 있도록 캘린더 뷰 전반에 '사람' 아이콘을 추가하고, 관련 상태를 업데이트하는 로직을 구현했습니다.

*   **기술적 분석:**
    *   `isShared` 속성을 `Event` 타입에 추가하여 일정의 공유 여부를 관리합니다.
    *   `useCalendarEvents` 훅에 `updateEventShared` 함수를 추가하여 개별 일정의 공유 상태를 동적으로 변경할 수 있게 했습니다.
    *   `CustomMonthEvent`, `CustomWeekEvent` 등 각 뷰 렌더러에 `people.svg` 아이콘을 삽입하였으며, 아이콘 추가에 따른 레이아웃 붕괴를 방지하기 위해 `EventTitle`의 너비를 조정했습니다.

```typescript
// src/shared/types/calendar/types.ts
export type Event = {
  // ... 기존 속성
  isShared?: boolean
}

// src/features/Calendar/hooks/useCalendarEvents.ts
const updateEventShared = useCallback((eventId: CalendarEvent['id'], isShared: boolean) => {
  setEvents((prev) =>
    prev.map((event) => (event.id === eventId ? { ...event, isShared } : event)),
  )
}, [])
```

### 2. 일정 편집 모달 상태 훅 분리 (작성자: Yeonjin Kim)
기존 `ItemEditorModal`에 집중되어 있던 복잡한 비즈니스 로직을 기능별 커스텀 훅으로 분리하여 코드 가독성과 재사용성을 높였습니다.

*   **기술적 분석:**
    *   `useEditorDraft`: 편집 중인 데이터의 초안(Draft) 상태를 관리합니다.

### 3. 공유 일정 검색 UI 구조 정리 (작성자: Yeonjin Kim)
공유 일정 설정 시 친구를 검색하고 추가하는 UI의 완성도를 높이고, 관련 유틸리티 함수를 정비했습니다.

*   **기술적 분석:**
    *   `MOCK_SCHEDULE_SHARE_FRIENDS`를 통해 API 연동 전 테스트 환경을 구축했습니다.
    *   `SearchFriend` 컴포넌트의 스타일을 개선하여 포커스 시의 시각적 피드백을 조정했습니다.
    *   `domVisibility.ts` 유틸리티를 추가하여 요소의 가시성 여부를 판단하는 로직을 공통화했습니다.
    *   `TitleSuggestionInput`에 컬러 커스터마이징 기능을 추가하여 UI 유연성을 확보했습니다.

```typescript
// src/shared/utils/scheduleShareFriend.ts
export const filterScheduleShareFriends = (
  friends: ScheduleShareFriend[],
  keyword: string,
): ScheduleShareFriend[] => {
  const trimmedKeyword = keyword.trim().toLowerCase()
  if (!trimmedKeyword) return []
  return friends.filter(
    (friend) =>
      friend.userName.toLowerCase().includes(trimmedKeyword) ||
      friend.email.toLowerCase().includes(trimmedKeyword),
  )
}

// src/shared/ui/scheduleTodo/SearchFriend/SearchFriend.style.ts
export const SearchInput = styled.input`
  &:focus {
    outline: none;
    border-color: #acacac;
    box-shadow: none;
  }
`
```
