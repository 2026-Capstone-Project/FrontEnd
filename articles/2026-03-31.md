### 1. 토스트 예외 처리 및 상태 관리 구조 개선 (작성자: Yeonjin Kim)
Yeonjin Kim 님은 리뷰 피드백을 반영하여 토스트 알림이 중복으로 발생하거나 누락되는 문제를 방지하기 위한 구조적 개선을 진행했습니다. 

**기술적 분석:**
- **상세 에러 추적:** `handledToastErrors`라는 `WeakSet`을 도입하여 이미 토스트로 처리된 에러 객체를 추적합니다. 이를 통해 동일한 에러가 상위 레이어로 전파될 때 중복 알림이 뜨는 것을 방지했습니다.
- **Zustand 최적화:** `ToastViewport`에서 `useShallow`를 적용하여 토스트 상태 변경 시 불필요한 리렌더링을 방지하고 성능을 최적화했습니다.
- **타입 안전성:** `ToastType`을 공통 타입 폴더(`src/shared/types/common/toast.ts`)로 분리하여 컴포넌트 간 의존성을 줄이고 타입 재사용성을 높였습니다.

```typescript
// src/shared/ui/common/Toast/ToastViewport.tsx 변경 사항
const ToastViewport = () => {
  const { toast, hideToast } = useToastStore(
    useShallow((state) => ({
      toast: state.toast,
      hideToast: state.hideToast,
    })),
  )

  if (!toast) return null;
  // ...
}
```

### 3. 에러 메시지 추출 유틸리티 및 폼 검증 강화 (작성자: Yeonjin Kim)
일정과 투두 폼 제출 시 발생하는 에러를 사용자 친화적인 메시지로 변환하는 로직을 강화했습니다.

**기술적 분석:**
- **재귀적 에러 메시지 수집:** `formError.ts` 내 `collectMessages` 함수를 구현하여, 복잡한 객체 구조를 가진 폼 에러(React Hook Form 등)에서 실제 출력해야 할 메시지만을 재귀적으로 추출합니다. 이때 `WeakSet`을 활용해 순환 참조로 인한 무한 루프를 방지했습니다.
- **네트워크 에러 대응:** Axios 에러 객체를 분석하여 `ERR_NETWORK`와 같은 저수준 에러에 대해서도 "네트워크 연결을 확인해 주세요"와 같은 구체적인 가이드를 제공하도록 개선했습니다.

```typescript
// src/shared/utils/error.ts 에러 메시지 처리 로직
export const getErrorMessage = (error: unknown, fallbackMessage = '잠시 후 다시 시도해주세요.') => {
  if (axios.isAxiosError<ErrorResponseData>(error)) {
    const responseMessage = error.response?.data?.message?.trim()
    if (responseMessage) return responseMessage

    if (error.code === 'ERR_NETWORK') {
      return '네트워크 연결을 확인한 뒤 다시 시도해주세요.'
    }
  }
  // ...
  return fallbackMessage
}
```

**시니어 개발자 총평:**
오늘 작업은 단순히 기능을 추가하는 것을 넘어, 서비스 전반의 **에러 대응 전략(Error Handling Strategy)**을 수립했다는 점에서 큰 의미가 있습니다. 특히 `WeakSet`을 활용한 에러 추적과 재귀적 폼 에러 추출 로직은 복잡한 프런트엔드 애플리케이션에서 발생할 수 있는 엣지 케이스를 훌륭하게 방어하고 있습니다. 향후 새로운 폼이나 API가 추가되더라도 일관된 사용자 경험을 제공할 수 있는 기반이 마련되었습니다.
