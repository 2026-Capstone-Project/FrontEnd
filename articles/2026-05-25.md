# [2026년 05월 25일] 캘린더 아키텍처 대규모 리팩터링 및 공유 일정 권한 제어 시스템 구축

오늘 팀 "Calio"는 캘린더 서비스의 완성도를 프로덕션 수준으로 끌어올리기 위해 대대적인 아키텍처 리팩터링과 신규 기능 개발을 단행했습니다. 기존에 비대해졌던 캘린더 컴포넌트의 상태와 비즈니스 로직을 다수의 전문화된 커스텀 훅으로 완벽히 분리하였으며, 공유 일정에 대한 소유권 기반의 읽기 전용(Read-Only) 권한 제어 시스템을 구축했습니다. 또한 실제 친구 검색 API를 연동하여 일정 공유 흐름을 고도화하고, 드래그 앤 드롭 및 렌더링 기하학 계산 로직을 모듈화하여 코드의 유지보수성과 성능을 동시에 확보한 하루였습니다.

### [Yeonjin Kim] 캘린더 비즈니스 로직의 훅 분리 및 공유 일정 권한 제어 고도화 (PR #99, #100)

Yeonjin Kim 개발자는 캘린더 엔진의 결합도를 낮추고 확장성을 극대화하기 위해 대규모 리팩터링을 주도했습니다. 복잡한 드래그 앤 드롭 수학식과 렌더링 레이아웃 계산을 분리하고, 비소유자의 수정을 방어하는 권한 제어 로직을 컴포넌트 전반에 구현했습니다.

#### 1. 캘린더 상태 관리 아키텍처 분할 (Custom Hooks 도입)
기존 `CustomCalendar.tsx`에 집중되어 있던 상태와 이벤트 핸들러들을 기능별 단일 책임 원칙(SRP)에 따라 커스텀 훅으로 완전히 파편화했습니다.
* **`useCustomCalendarController`**: 캘린더의 전반적인 뷰, 날짜 범위, 이벤트 데이터를 총괄 제어합니다.
* **`useCalendarDateCellWrapper`**: 날짜 셀 클릭 및 슬롯 선택 브릿지 역할을 수행합니다.
* **`useCalendarDayViewTiming` / `useCalendarTodoTimingPatch`**: 일간 뷰 및 투두의 시간 보정 및 패치 로직을 캡슐화합니다.
* **`useCalendarDeleteConfirm` / `useCalendarRecurringDropConfirm`**: 반복 일정의 수정/삭제 시 발생하는 모달 컨텍스트 상태를 분리 관리합니다.
* **`useCalendarSelectionBridge`**: 모달 에디터와 캘린더 선택 영역 간의 상태 동기화를 중재합니다.

#### 2. 에디터 폼 라이프사이클 공통화 (`useEditorFormLifecycle`)


#### 3. 소유권 기반 읽기 전용(Read-Only) 권한 제어 시스템 구축
공유 일정 중 본인이 소유하지 않은 일정(`isOwner: false`)에 대해 수정을 원천 차단하는 방어적 UX를 구현했습니다. 이를 위해 입력 및 선택 컴포넌트에 `readOnly` 속성과 비정상 접근을 감지하는 `onReadOnlyAttempt` 콜백을 주입했습니다.
* 대상 컴포넌트: `TitleSuggestionInput`, `CustomTimePicker`, `Checkbox`, `SelectColor`, `ScheduleDateTimeSection`, `ScheduleDetailsSection`
* 소유자가 아닐 경우 폼 필드가 비활성화되며, 클릭 시 "수정 권한이 없습니다" 등의 피드백(Toast)을 유도할 수 있도록 설계했습니다.

#### 5. 실시간 친구 검색 API 연동 및 성능 최적화
공유 일정에 친구를 초대하기 위해 기존의 목업 데이터를 제거하고 실제 백엔드 API `/friends/search`를 연동했습니다. 검색어 입력 시 불필요한 API 호출을 방지하기 위해 `useThrottledValue` 훅을 적용하여 디바운싱/스로틀링 처리를 완료했습니다.

```typescript
// src/shared/api/friend/api.ts
import axiosInstance from '../axios'
import type { TCommonResponse } from '@/shared/types/common/common'
import type { FriendDetail } from '@/shared/types/friend/types'

export const getFriendSearch = async (
  keyword: string,
): Promise<TCommonResponse<{ friendDetailList: FriendDetail[] }>> => {
  const { data } = await axiosInstance.get('/friends/search', {
    params: { keyword },
  })
  return data
}
```

#### 6. 스타일 및 환경 설정 미세 조정 (Bug Fixes)
* **Flexbox 레이아웃 붕괴 방지**: 캘린더 내부의 원형 마커(`Circle`) 및 투두 체크박스(`TodoCheckbox`)가 텍스트 길이에 의해 찌그러지는 현상을 방지하기 위해 `flex: 0 0 auto` 속성을 추가했습니다.

```typescript
// src/features/Calendar/components/CustomEvent/CustomEvent.style.ts
export const Circle = styled.div<{ backgroundColor?: string }>`
  min-width: 10px;
  min-height: 10px;
  flex: 0 0 auto; /* 줄어듦 방지 */
  border-radius: 50%;
  background-color: ${({ backgroundColor }) => backgroundColor ?? 'transparent'};
`
```

* **경로 에일리어스 설정 보완**: `tsconfig.app.json`에서 빌드 시 에일리어스 경로 해석 오류를 방지하기 위해 `baseUrl` 설정을 정규화했습니다.

```json
// tsconfig.app.json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```
