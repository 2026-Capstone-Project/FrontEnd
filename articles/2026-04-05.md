### [Yeonjin Kim] 캘린더 컴포넌트 리팩터링 및 시간 설정 로직 개선
복잡해진 `CustomCalendar` 컴포넌트를 `Dialogs`, `MobileActions` 등으로 분리하여 유지보수성을 높였습니다. 특히 `useCalendarDraftEvent` 훅을 신설하여 일정 생성 중의 상태 관리를 추상화하였고, 일정 생성 시 기본 종료 시간을 설정하는 로직을 보완하여 데이터 정합성을 높였습니다.

```typescript
// src/shared/hooks/form/useScheduleFormFields.ts
const getDefaultEndDate = (defaultStart: Date, initialEvent?: CalendarEvent | null) => {
  if (initialEvent?.end && initialEvent.end !== initialEvent.start) {
    return new Date(initialEvent.end)
  }
  if (initialEvent?.start) {
    return new Date(defaultStart)
  }
  return new Date(defaultStart.getTime() + 60 * 60 * 1000) // 기본 1시간 추가 로직 보완
}
```

### [Yeonjin Kim] 사파리 브라우저 100vh 대응 및 인증 루프 방지
iOS 사파리 등 모바일 브라우저에서 주소창 높이로 인해 레이아웃이 깨지는 문제를 해결하기 위해 `vh` 단위 대신 `dvh`(Dynamic Viewport Height)를 도입했습니다. 또한, 인증 쿠키가 만료되었을 때 리프레시 토큰 요청이 무한히 반복되는 현상을 막기 위해 `refreshBlocked` 플래그를 도입한 인증 복구 로직을 구현했습니다.

```typescript
// src/shared/layout/MainLayout.tsx
const Wrapper = styled.div`
  min-height: 100dvh;
  height: 100dvh;
  max-height: 100dvh;
  display: flex;
  flex-direction: column;
  width: 100%;
`
```

### [Yeonjin Kim] CSRF 토큰 처리 최적화 및 보안 헤더 추가
기존에 쿠키에서 직접 값을 읽어오던 방식에서 서버의 특정 엔드포인트를 통해 CSRF 토큰을 안전하게 갱신하는 `fetchCsrfToken` 방식으로 개선했습니다. 이를 통해 토큰 재발급(Refresh) 요청 시에도 최신 CSRF 헤더를 포함할 수 있게 되어 보안 수준이 한층 강화되었습니다.

```typescript
// src/shared/api/axios.ts
const fetchCsrfToken = async () => {
  const { data } = await axios.get(`${import.meta.env.VITE_SERVER_URL}/security/csrf`, {
    withCredentials: true,
  })
  const resultString = data.result || ''
  return resultString
}

// 인증 복구 상태 초기화 로직
export const resetAuthRecoveryState = () => {
  refreshBlocked = false
}
```
