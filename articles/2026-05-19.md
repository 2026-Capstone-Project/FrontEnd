### [Yeonjin Kim] 일정 수정 모달의 변경 감지(Unsaved Changes) 및 공유 로직 개선
사용자가 내용을 수정 중 페이지를 이탈하려고 할 때 경고를 띄우는 `useUnsavedCloseGuard`를 개선했습니다. `useLocation`을 활용해 경로 변경을 감지하고, 이탈 시점의 경로를 저장하여 사용자 확인 후 이동할 수 있도록 처리했습니다. 또한, 공유 친구 목록에서 중복된 데이터가 표시되지 않도록 `dedupeFriends` 유틸리티를 추가했습니다.

```typescript
// src/shared/ui/Modals/ScheduleEditor/ShareSchedulePanel.tsx
const dedupeFriends = (friends: DisplayedFriend[]) => {
  const seenIds = new Set<string>()
  return friends.filter((friend) => {
    if (seenIds.has(friend.id)) return false
    seenIds.add(friend.id)
    return true
  })
}
```

### [Yeonjin Kim] 일정 수정 API 및 데이터 모델 정제
일정 수정 시 `friendIds`가 `null`로 전송될 수 있도록 타입을 완화하고, 참여자 정보에서 실제 친구 ID를 추출하는 `getEventFriendIds` 유틸리티를 구현하여 API 통신의 정확도를 높였습니다.

```typescript
// src/shared/utils/eventParticipants.ts
export const getEventFriendIds = (event?: CalendarEvent | null) => {
  if (event?.friendIds != null) return event.friendIds

  return (
    event?.eventParticipantInfo
      ?.map(getEventParticipantFriendId)
      .filter(Boolean) ?? []
  )
}
```
