# [2026년 05월 31일] 친구 관리 시스템 출시 및 캘린더·공유 일정 아키텍처 대규모 고도화

오늘 팀 "Calio"는 서비스의 소셜 확장성을 확보하기 위한 **'친구 관리' 기능의 UI/UX 및 API 연동을 완료**하고, 그동안 `develop` 브랜치에서 다듬어온 **캘린더 엔진 및 공유 일정 시스템의 대규모 리팩토링 성과를 `main` 브랜치에 성공적으로 병합(Deploy)**했습니다. 

이번 작업을 통해 사용자들은 친구를 검색하고 요청을 주고받을 수 있을 뿐만 아니라, 일정에 친구를 초대하여 실시간으로 공유 일정을 시각화하고 권한(소유자/참여자)에 따라 안전하게 관리할 수 있게 되었습니다. 또한 빌드 안정성을 저해하던 패키지 누락 및 경로 설정 오류를 해결하여 프로덕션 배포의 완성도를 극대화했습니다.

### 2. 친구 페이지 UI/UX 구축 및 소셜 API 연동 (작성자: Yujin Ji)
**Yujin Ji** 개발자는 사용자들이 소셜 관계를 맺고 관리할 수 있는 '친구 페이지'를 신규 구축하고 백엔드 API와의 실시간 데이터 동기화를 완료했습니다.

*   **반응형 레이아웃 설계**: `Friend.styles.ts`를 통해 데스크톱과 모바일 환경에 유연하게 대응하는 Flex 기반 2열 레이아웃(`PageLayout`, `Column`)을 구현했습니다.
*   **컴포넌트 모듈화**: 친구 목록을 렌더링하는 `FriendItem`, 섹션별 그룹화를 담당하는 `FriendListSection`, 그리고 공유된 일정을 직관적으로 보여주는 `ScheduleItem` 및 `SharedScheduleItem`을 설계했습니다.
*   **React Query 기반 상태 관리**: `useFriendMutations` 훅을 신규 도입하여 친구 요청 수락(`acceptRequest`), 거절, 삭제 시 관련 쿼리 키를 무효화(`invalidateQueries`)함으로써 화면에 즉각적으로 변경 사항이 반영되도록 처리했습니다.
*   **내비게이션 통합**: 전역 `Header`에 '친구' 메뉴를 추가하고 `MainRoutes`에 `/friends` 경로를 매핑하여 서비스 접근성을 높였습니다.

```typescript
// src/shared/hooks/friends/useFriendsMutations.ts 일부
import { useMutation, useQueryClient } from '@tanstack/react-query'
import { friendApi, friendRequestApi } from '@/shared/api/friends/friends'

export function useFriendMutations() {
  const queryClient = useQueryClient()

  // 친구 요청 수락 Mutation
  const acceptMutation = useMutation({
    mutationFn: (friendRequestId: number) => friendRequestApi.acceptRequest({ friendRequestId }),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['friends'] })
      queryClient.invalidateQueries({ queryKey: ['friendRequests', 'received'] })
    }
  })
  
  return { acceptMutation }
}
```

### 1. 캘린더 엔진 리팩토링 및 비즈니스 로직 커스텀 훅 분리 (작성자: Yujin Ji)
기존에 `CustomCalendar.tsx` 컴포넌트에 비대하게 집중되어 있던 상태 관리와 이벤트 핸들링 로직을 단일 책임 원칙(SRP)에 따라 각각의 전용 커스텀 훅으로 완전 분리했습니다.

*   **`useCustomCalendarController`**: 캘린더의 현재 날짜, 뷰 모드, 이벤트 필터링 상태를 총괄 제어합니다.
*   **`useCalendarDragDrop` & `useCalendarDayViewTiming`**: 드래그 앤 드롭을 통한 일정 시간 변경, 일간 뷰(Day View)에서의 미세 시간 슬롯 계산 및 유효성 검증 로직을 캡슐화했습니다.
*   **`useCalendarTodoActions` & `useCalendarDeleteConfirm`**: 할 일 완료 토글 처리 및 반복 일정 삭제 시의 범위 선택 모달 제어 로직을 컴포넌트 외부로 격리하여 코드 가독성을 극대화했습니다.


### 2. 공유 일정(Shared Schedule) 시각화 및 권한 제어 시스템 구축 (작성자: Yujin Ji)
협업과 공유를 위한 핵심 스펙인 '공유 일정'의 데이터 모델을 정립하고, 사용자 역할에 따른 보안 및 UI 제어 로직을 이식했습니다.

*   **데이터 모델 확장**: `CalendarEvent` 타입에 `isShared`, `isOwner`, `eventParticipantInfo` 필드를 추가하여 일정의 소유권과 참여자 목록을 추적할 수 있도록 했습니다.
*   **공유 식별자 UI 반영**: 공유된 일정일 경우 캘린더 뷰 내에서 직관적으로 인지할 수 있도록 사람 아이콘(`people.svg`)이 표시되도록 `CustomMonthEvent`, `CustomWeekEvent`를 개선했습니다.
*   **Read-Only 권한 제어**: 일정 상세 에디터 진입 시, 소유자(`isOwner: false`)가 아닌 참여자일 경우 모든 입력 필드와 컬러 선택기(`SelectColor`), 시간 선택기(`CustomTimePicker`)를 `readOnly` 및 비활성화 상태로 렌더링하고 수정 시도를 원천 차단하도록 방어적 프로그래밍을 적용했습니다.

```typescript
// src/shared/types/calendar/types.ts 중 확장된 Event 타입 정의
export type Event = {
  id: number
  title: string
  start: string
  end: string
  isAllDay: boolean
  color: EventColorType
  recurrenceGroup: RecurrenceGroup | null
  isShared?: boolean              // 공유 일정 여부
  isOwner?: boolean               // 현재 사용자가 소유자인지 여부
  friendIds?: number[] | null     // 초대된 친구 ID 목록
  eventParticipantInfo?: Array<{  // 참여자 상세 정보
    eventParticipantId: number
    friendId?: number
    email: string
    name: string
  }>
}
```

### 3. 반복 주기(Recurrence Scope) 상수화 및 공통 폴더 구조 재편 (작성자: Yujin Ji)
코드베이스 전반의 유지보수성을 향상시키기 위해 하드코딩된 문자열을 제거하고, 산재해 있던 컴포넌트들의 물리적 위치를 재정렬했습니다.

*   **반복 범위 상수화 (`recurrenceScope.ts`)**: 일정과 할 일의 반복 수정 범위를 결정하는 옵션들을 상수로 정의하여 타입 안정성을 확보했습니다.
    *   `RECURRENCE_EVENT_SCOPE`: `THIS_EVENT` (이 일정만), `THIS_AND_FOLLOWING_EVENTS` (이후 모든 일정)
    *   `RECURRENCE_TODO_SCOPE`: `THIS_TODO`, `THIS_AND_FOLLOWING`
*   **`scheduleTodo` 공통 폴더 신설**: 일정과 할 일 모달에서 공통으로 사용되던 `RepeatPanel`, `RepeatTypeGroup`, `TitleSuggestionInput`, `SearchPlace` 등의 컴포넌트들을 `src/shared/ui/scheduleTodo/` 경로로 일원화하여 중복 코드를 제거하고 import 관계를 단순화했습니다.
